# workflow

The **operator's** working notes for directing agents: task briefs in `task/` (one directory per ticket — `task/<TICKET>/<TICKET>.md` with status/done/next plus related artifacts; see the TASK-123 example), accumulated notes in `notes/`. These files are personal drafts — often stale, half-finished, or tied to a past task, so treating them as truth will mislead you. Agents: never read or use anything here on your own initiative; only open a file the user explicitly points you to, and trust it only for that task. Sole exception: `task/<TICKET>/progress_log.md` — when working on that ticket, read it to see where the task stands and keep it briefly updated (create the directory and the file if the ticket has none); the rest of the directory stays opt-in.
